## Phase 7 Detailed Design (2026-01-06)

### Current State Analysis

**Realtime Server components**:
- `server.go` - Server struct holds `engine.Service`, uses `WatchCollection()` for events
- `hub.go` - Hub broadcasts `storage.Event`, does local CEL matching per client
- `client.go` - Client compiles CEL filters locally via `compileFiltersToCEL()`
- `cel.go` - Local CEL compilation (to be deleted, moved to streamer)

**Key Types**:
```go
// storage.EventType (string): "create", "update", "delete"
type EventType string
const (
    EventCreate EventType = "create"
    EventUpdate EventType = "update"
    EventDelete EventType = "delete"
)

// streamer.OperationType (int): OperationInsert, OperationUpdate, OperationDelete
type OperationType int
const (
    OperationUnspecified OperationType = iota
    OperationInsert   // maps to "create"
    OperationUpdate   // maps to "update"
    OperationDelete   // maps to "delete"
)

// streamer.EventDelivery - what Hub receives from Streamer
type EventDelivery struct {
    SubscriptionIDs []string   // Which subscriptions matched
    Event           *Event     // The actual event
}
```

### Refactoring Strategy

**Phase 7.1: Update server.go**
```go
type Server struct {
    hub            *Hub
    queryService   engine.Service    // Keep for Pull() snapshot
    streamer       streamer.Service  // NEW: for event streaming
    dataCollection string
    auth           identity.AuthN
    cfg            Config
}

func (s *Server) StartBackgroundTasks(ctx context.Context) error {
    go s.hub.Run(ctx)

    // NEW: Use Streamer instead of WatchCollection
    stream, err := s.streamer.Stream(ctx)
    if err != nil {
        return err
    }

    go func() {
        for {
            delivery, err := stream.Recv()
            if err != nil {
                return
            }
            s.hub.BroadcastDelivery(delivery)  // NEW method
        }
    }()
    return nil
}
```

**Phase 7.2: Update hub.go**
- Change `broadcast chan storage.Event` ‚Üí `broadcast chan *streamer.EventDelivery`
- Add `BroadcastDelivery(*streamer.EventDelivery)` method
- Route by `delivery.SubscriptionIDs` instead of local CEL matching
- Add type mapping: `streamer.OperationType` ‚Üí `storage.EventType`

**Phase 7.3: Update client.go**
- Add `stream streamer.Stream` field to Client
- On Subscribe: call `stream.Subscribe(tenant, collection, filters)`
- On Unsubscribe: call `stream.Unsubscribe(subID)`
- Remove `compileFiltersToCEL()` call (CEL now in Streamer)

**Phase 7.4: Delete cel.go**
- All 106 lines moved to `internal/streamer/internal/cel/`

**Phase 7.5: Type Mapping Function**
```go
func operationToEventType(op streamer.OperationType) storage.EventType {
    switch op {
    case streamer.OperationInsert:
        return storage.EventCreate
    case streamer.OperationUpdate:
        return storage.EventUpdate
    case streamer.OperationDelete:
        return storage.EventDelete
    default:
        return ""
    }
}
```

### Key Considerations

1. **Subscription Sync**: Each WebSocket Client needs its own Stream, OR Hub manages a single Stream and maps SubscriptionIDs to Clients
2. **SSE Clients**: Currently subscribe via query params, need to sync to Streamer
3. **Snapshot**: Still uses `queryService.Pull()`, no change needed
4. **Backward Compatibility**: Protocol messages to SDK unchanged (TypeEvent, TypeSubscribeAck, etc.)

## 1. Objective

Replace `internal/csp` with a new Streamer service that provides:
- **Centralized fan-out**: O(1) DB load via single Puller stream
- **Centralized CEL matching**: Match once, route to many Gateways
- **Clean proto contracts**: No `storage.Event` dependency in realtime layer

---

## 2. Architecture

### Before (CSP)
```
Realtime Server ‚îÄ‚îÄ> engine.WatchCollection() ‚îÄ‚îÄ> csp.Watch() ‚îÄ‚îÄ> MongoDB
       ‚îÇ
       ‚îî‚îÄ‚îÄ Hub does local CEL matching per client
```
**Problems**: Connection explosion, duplicate matching, tight coupling

### After (Streamer)
```
MongoDB ‚îÄ‚îÄ> Puller ‚îÄ‚îÄgRPC‚îÄ‚îÄ> Streamer ‚îÄ‚îÄgRPC‚îÄ‚îÄ> Realtime Server (Gateway)
                                ‚îÇ                       ‚îÇ
                                ‚îî‚îÄ‚îÄ CEL matching here   ‚îî‚îÄ‚îÄ Pre-matched events
```

---

## 3. Key Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Primary consumer | `internal/api/realtime/` | Realtime Server is actual CSP consumer, not Engine |
| D2 | CEL location | `internal/streamer/internal/cel/` | Centralized matching in Streamer |
| D3 | Event types | Define `StreamerEvent` in proto | No dependency on `internal/storage` |
| D4 | Migration | Direct cutover | CSP is internal, no external consumers |
| D5 | Puller dependency | Accept `storage.StoredDoc` in Puller | Streamer converts to proto at Gateway boundary |
| D6 | CEL data format | Decode JSON ‚Üí map in Streamer | `puller.ChangeEvent.FullDocument.Data` already a map |
| D7 | Operation type mapping | `INSERT‚Üícreate`, `UPDATE‚Üíupdate`, `DELETE‚Üídelete` | Map proto enum to client JSON |
